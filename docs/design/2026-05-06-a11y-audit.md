# Helm UI Accessibility Audit — 2026-05-06

> WCAG 2.1 AA. Companion to `2026-05-06-polish-pass.md`. Findings here that overlap with the polish pass are marked `(see P0-N)` and folded into the same fix.

## Summary

Scope: every file under `web/src/`. **9 issues total — 4 critical, 4 major, 1 minor.** Dominant failure modes: missing focus indicators, mid-contrast secondary text in dark mode, unlabeled form controls, no `prefers-reduced-motion` handling for the polish-pass animations.

## Perceivable

| # | Issue | WCAG | Severity | Fix |
|---|---|---|---|---|
| A1 | `--text-secondary: #8e8e93` on dark `--bg-elev: #2c2c2e` measures ≈4.0:1 — below 4.5:1 required for normal text. Affects every `.muted` paragraph and `.label` uppercase block in dark mode. | 1.4.3 Contrast | 🔴 Critical | Bump dark `--text-secondary` to `#a1a1a6` (≈4.8:1) |
| A2 | `<pre>` background and card background nearly identical in dark mode (1.04:1 surface contrast); the contained code text is fine vs background but the *boundary* of the code block is invisible — fails 1.4.11 non-text contrast for UI components. | 1.4.11 Non-text contrast | 🔴 Critical | Same fix as polish-pass P0-6: dark `.helm-card pre` background becomes `rgba(0,0,0,0.25)` |
| A3 | Status dots `--success` / `--warn` / `--danger` against card background in dark — `#34c759` on `#2c2c2e` is ≈4.6:1 ✓; `#ff9500` is ≈4.0:1 ✗ for non-text contrast since the dot is a meaningful UI component. | 1.4.11 | 🟡 Major | Add `box-shadow: 0 0 0 2px <color>22` halo around dot (also polish-pass P2-3) — restores boundary contrast in dark |
